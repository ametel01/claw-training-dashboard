

## Gym dashboard + training system updates (durable)
- User preference: concise responses; avoid verbose logs in dashboard presentation.
- Dashboard now served persistently via macOS LaunchAgent `com.brunoclaw.gym-dashboard` and exposed through Tailscale HTTPS serve (`https://brunos-macbook-air.tail8a86b0.ts.net/dashboard/`).
- Added dashboard capabilities:
  - interactive day modal from both Daily Tiles and Week Progress,
  - planned vs completed delta card with substitution highlight,
  - next-session suggestion block,
  - week/cycle lock banner (e.g., Leader Week N + active percentages),
  - compact “Today at a glance” card,
  - cardio analytics (Z2 compliance + VO2 trend + Z2 HR trend),
  - audit log panel for config/override changes,
  - session action buttons (Main done / Supplemental modified / Cardio done),
  - cardio logging now prompts for avg HR.
- Training logic updated per user rule:
  - Main week percentages: W1 65/75/85, W2 70/80/90, W3 75/85/95.
  - Supplemental BBS percentages: W1 65%, W2 70%, W3 75%.
- Cardio plan adjustments:
  - Global Z2 duration set to 30 min.
  - Saturday VO2 1-min hard speed set to 15.0 km/h; easy segment adjusted to 1.5 min.
- Current-week schedule shift preference (one-off): Monday off, all planned work moved forward by one day only for current week. Prior-week accidental override was removed.
- Bodyweight baseline for strength-level calculations set to 85 kg; dashboard shows estimated 1RM, strength level, and next-level trigger weight.
- Readiness/pain framework introduced in DB (`recovery_status`: green/yellow/red) and reflected in dashboard planning/suggestions.


- Dashboard cardio analytics: added Z2-only HR variation graph (last 12 weeks) and fixed missing-data bug by falling back to `max_hr` when `avg_hr` is null for older Z2 entries.
- Cardio logging workflow now prompts for avg HR on "Cardio done" and persists to `cardio_sessions.avg_hr`.
- Session action UX update per user request: renamed button label/action to "Supplemental done" (from modified) and added "Rings done" action/button; server endpoint now supports `supp_done` and `rings_done`.
- Added audit log infrastructure in DB (`audit_log` table + triggers on `config` and `schedule_overrides`) and dashboard panel to display recent rule/override changes with timestamp and old→new values.
